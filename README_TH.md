## 🛡️ TESA Defense - ระบบตรวจจับและติดตามโดรน

### ภาพรวม
ระบบที่ก้าวหน้าในการมองเห็นด้วยคอมพิวเตอร์และแมชชีนเลิร์นนิงสำหรับตรวจจับ ติดตาม และประมาณพิกัด GPS ของโดรนขนาดเล็กจากวิดีโอ ระบบนี้รวมการตรวจจับวัตถุ YOLO เข้ากับการประมาณพิกัด GPS ตามแมชชีนเลิร์นนิง

### คุณสมบัติหลัก
- **🎯 การตรวจจับโดรนที่แม่นยำ**: ใช้พลัง YOLO v8 (`best_6.1.pt`)
- **📍 การประมาณพิกัด GPS แบบเรียลไทม์**: แปลงค่ากล่องขอบเขต (Bounding Box) เป็นพิกัด GPS (ละติจูด/ลองจิจูด/ความสูง)
- **👁️ ระบบติดตามที่ชาญฉลาด**: ติดตามโดรนได้ถึง 2 ลำพร้อมกัน มี ID เสถียร และการเห็นภาพวิถีการบิน
- **🎬 เอาต์พุตวิดีโอ**: สร้างไฟล์ `.mp4` พร้อมเวลา ข้อมูลการตรวจจับ และวิถีการบิน
- **⚡ สลับ GPU/CPU อัตโนมัติ**: ใช้ประโยชน์จากการประมวลผล CUDA เมื่อมีการใช้ได้
- **🔗 การรวมเข้ากับ TESA API**: ส่งข้อมูลแบบเรียลไทม์ผ่าน REST API

## 🚁 TESA Offense - ระบบควบคุมโดรนควอดโรเตอร์

### ภาพรวม
ระบบ MATLAB/Simulink สำหรับจำลองพลวัตโดรนควอดโรเตอร์ การวางแผนวิถี และการควบคุม นำไปใช้ควบคุมการติดตามทางเรขาคณิตโดยใช้ตัวควบคุม PD สำหรับการบินอัตโนมัติ

### คุณสมบัติหลัก
- **🔧 จำลองพลวัตโรเตอร์**: แบบจำลองทางกายภาพของโดรนควอดโรเตอร์ที่สมบูรณ์

- **🎮 ระบบควบคุม**:
  - ควบคุมการติดตามทางเรขาคณิต (GTC)
  - ตัวควบคุม PD สำหรับแรงโน้มถ่วงและควบคุมแรงขับ
  - การรวมเข้ากับตัวควบคุมความเร็วอิเล็กทรอนิกส์ (ESC)
- **⚡ การจำลองตามสมการอนุพันธ์ลำดับที่ 1 ขึ้นไป**: พลวัตลำดับแรกและลำดับที่สูงกว่า

### สถาปัตยกรรมระบบ

#### สถาปัตยกรรมการควบคุม
```
จุดรอบเส้นทาง → การวางแผนวิถี → ควบคุมการติดตามทางเรขาคณิต → ตัวควบคุมท่าทาง → 
คำสั่งมอเตอร์ → ESC → แรงขับโรเตอร์ → พลวัต → ข้อมูลป้อนกลับ
```

#### พลวัตของโดรนควอดโรเตอร์
- **แบบจำลองโรเตอร์**: พลวัตมอเตอร์แต่ละตัวพร้อมแรงขับเป็นสัดส่วนกับกำลังสองของความเร็วหมุนวน
- **พลวัตท่าทาง**: จลนศาสตร์มุมออยเลอร์และพลวัตของอัตราเชิงมุม
- **พลวัตตำแหน่ง**: สมการการเคลื่อนไหวเชิงเส้นภายใต้แรงโน้มถ่วง
- **ข้อมูลป้อนกลับ**: สถานะที่สมบูรณ์ (ตำแหน่ง ความเร็ว ท่าทาง อัตราเชิงมุม) พร้อมใช้งานสำหรับการควบคุม

### สมการการควบคุมหลัก

#### 1. ควบคุมการติดตามทางเรขาคณิต
```
Track​ = [-k_e·e - k_ė·ė]

โดยที่:
  e = ข้อผิดพลาดของตำแหน่ง
  ė = ข้อผิดพลาดของความเร็ว
  k_e, k_ė = ค่ากำไรการควบคุม
```

#### 2. พลวัตโรเตอร์
```
L·f = L·τ + ω·(ω × I·ω) + (ld m·g_earth)

โดยที่:
  L = เมทริกซ์การแก้ไขข้อผิดพลาดท่าทาง
  f = แรงขับแบบรวม
  τ = คำสั่งแรงบิด
  ω = ความเร็วเชิงมุม
```

#### 3. วิถีการไหลแต่นิ่มน้อยที่สุด
- สร้างพหุนามลำดับที่ 8 ที่เรียบเนียน
- ทำให้เป็นศูนย์ความเร่งที่จุดรอบเส้นทาง
- เพิ่มประสิทธิภาพสำหรับอนุพันธ์ขั้นต่ำ (snap)

### การรันระบบ

#### ตั้งค่าพื้นฐาน
1. เปิด MATLAB
2. นำทางไปยัง `Offense-Day1-Problem` หรือ `Offense-Day2-Problem`
3. เรียกใช้สคริปต์ตั้งค่าพื้นที่ทำงาน:
   ```matlab
   TESA_Workspace1
   % หรือ
   TESA_Workspace2
   ```

#### การจำลองใน Simulink
1. เปิดแบบจำลอง: `quadrotorsmodel2.slx`
2. ตั้งค่าจุดรอบเส้นทางในสคริปต์พื้นที่ทำงาน:
   ```matlab
   waypoints = [0 0 0 0;      % x y z yaw
                0 0 -10 1;
                10 10 -10 1;
                10 30 -20 1;
                30 40 -30 1];
   timepoints = [0 5 10 15 20];  % เวลาที่แต่ละจุด (วินาที)
   ```
3. เรียกใช้การจำลองและดูผลลัพธ์

#### การวางแผนวิถี
```matlab
% สร้างค่าสัมประสิทธิ์การไหลแต่นิ่มน้อยที่สุด
trajectory_coef = min_snap_Coef(waypoints, timepoints);

% สอบถามวิถีในเวลาที่กำหนด
[position, velocity, acceleration] = getTrajectory(trajectory_coef, t);
```
